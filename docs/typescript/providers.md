# Providers

Spectra provides built-in LLM providers, a provider registry for custom ones, and an auto-generated model catalog covering thousands of models across 150+ providers.

## Built-in Providers

```typescript
import { initProviders } from "@mohanscodex/spectra-ai";

initProviders(); // auto-called on import — registers all built-in providers
```

## Model Registry

Spectra maintains a comprehensive, **auto-generated** model catalog at build time. The file `src/models.ts` is generated by `scripts/generate-models.ts` during `bun run build`.

### How Models Are Generated

The build script fetches model data from two sources:

1. **OpenRouter API** (`https://openrouter.ai/api/v1/models`) — Filters to models that support `"tools"` in their `supported_parameters`. This captures models from dozens of providers that are listed on OpenRouter.

2. **models.dev API** (`https://models.dev/api.json`) — A community-maintained model database. Only models with `tool_call: true` are included.

The script merges both sources, deduplicates by provider + ID, categorizes OpenRouter models by prefix (with a mapping table for aliases like `together` → `togetherai`), and outputs a sorted `Record<string, ModelEntry[]>` structure.

### Generated Output

```typescript
// Auto-generated by scripts/generate-models.ts
// Do not edit manually

export interface ModelEntry {
  id: string;
  name: string;
}

const MODELS: Record<string, ModelEntry[]> = {
  "302ai": [/* ... */],
  anthropic: [/* ... */],
  deepseek: [/* ... */],
  // 150+ providers, 4000+ models
};

export function getProviderModels(providerId: string): ModelEntry[] {
  return MODELS[providerId] || [];
}
```

The generated file typically contains **4,000+ models** across **150+ providers**. Top providers include Anthropic, OpenAI, Google, Meta, Mistral, and dozens of hosting platforms.

### Accessing Models

```typescript
import { getProviderModels, getModels, listProviders } from "@mohanscodex/spectra-ai";

// From the static catalog
const anthropicModels = getProviderModels("anthropic");
// [{ id: "claude-sonnet-4-20250514", name: "Claude Sonnet 4" }, ...]

// From a registered provider (delegates to listModels if available)
const models = await getModels("anthropic");

// List all registered provider keys
const providers = listProviders();
// ["anthropic", "openai-completions", "openai-responses", "groq", "openrouter", "xai", ...]
```

### Provider `listModels` Integration

Each built-in provider has an optional `listModels()` function. The OpenAI-compatible providers delegate to the generated model catalog:

```typescript
// In register-builtins.ts
wrapOpenAIProvider("deepseek", "https://api.deepseek.com/v1")
// listModels: () => import("../models.js").then((m) => m.getProviderModels("deepseek"))
```

The **OpenRouter** provider is special — it first attempts to fetch live models from the OpenRouter API at runtime, falling back to the static catalog if the fetch fails.

### Regenerating Models

```bash
# Regenerate during build (automatic)
bun run build

# Regenerate manually
bun run generate-models
```

The script requires network access to fetch from OpenRouter and models.dev. If either source is unavailable, it logs a warning and continues with whatever data it could retrieve.

## Using a Provider

```typescript
const agent = new Agent({
  model: {
    id: "claude-sonnet-4-20250514",
    name: "Claude Sonnet 4",
    provider: "anthropic",
    api: "anthropic-messages",
  },
});
```

### Model Object

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | `string` | Yes | Model identifier (e.g., `gpt-4o`, `claude-sonnet-4-20250514`) |
| `name` | `string` | Yes | Display name |
| `provider` | `string` | Yes | Provider registry key (must match a registered provider) |
| `api` | `string` | Yes | API type within the provider |
| `baseUrl` | `string` | No | Override API endpoint |
| `reasoning` | `boolean` | No | Enable reasoning mode |
| `maxTokens` | `number` | No | Max tokens in response |
| `headers` | `Record<string, string>` | No | Custom headers |

## API Keys

### Environment Variables (Recommended)

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
export OPENAI_API_KEY="sk-..."
```

The providers automatically read these from `process.env`.

### Dynamic Resolution

```typescript
const agent = new Agent({
  model,
  getApiKey: async (provider) => {
    if (provider === "openai") return process.env.OPENAI_API_KEY;
    if (provider === "anthropic") return process.env.ANTHROPIC_API_KEY;
    return undefined;
  },
});
```

### Per-Request Key

```typescript
const agent = new Agent({
  model,
  streamOptions: { apiKey: "sk-custom-key" },
});
```

::: warning
Never hardcode API keys in source code. Use environment variables, a secrets manager, or a dynamic resolution function.
:::

## Custom Providers

Register any LLM provider that supports streaming:

```typescript
import { registerProvider } from "@mohanscodex/spectra-ai";

registerProvider({
  name: "my-provider",
  stream: (model, context, options) => {
    const stream = new AssistantMessageEventStream();

    // Your streaming implementation here
    // Push events as they arrive:
    stream.push({
      type: "content_delta",
      delta: { type: "text", text: "Hello " },
    });

    // Signal completion:
    stream.end({
      content: [{ type: "text", text: "Hello world" }],
      stopReason: "stop",
    });

    return stream;
  },
});
```

### OpenAI-Compatible Endpoints

For providers that implement the OpenAI API format (Groq, Together, local models):

```typescript
registerProvider({
  name: "groq",
  stream: (model, context, options) => {
    const client = new OpenAI({
      apiKey: options?.apiKey || process.env.GROQ_API_KEY,
      baseURL: "https://api.groq.com/openai/v1",
    });

    const stream = new AssistantMessageEventStream();

    (async () => {
      const response = await client.chat.completions.create({
        model: model.id,
        messages: context.messages,
        stream: true,
      });

      for await (const chunk of response) {
        const text = chunk.choices[0]?.delta?.content;
        if (text) {
          stream.push({
            type: "content_delta",
            delta: { type: "text", text },
          });
        }
      }

      stream.end({
        content: [{ type: "text", text: "full response" }],
        stopReason: "stop",
      });
    })();

    return stream;
  },
});
```
