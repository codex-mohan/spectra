**Problem:** The maturin build pipeline is broken in two ways:
1. `pyproject.toml` is missing `manifest-path` under `[tool.maturin]` — maturin can't find the Cargo.toml because it's not in the Python package directory (it's at `../../crates/spectra-pyo3/Cargo.toml`)
2. `Cargo.toml` `[lib]` is missing `name = "spectra_pyo3"` — maturin needs to know the lib name, and with a dash-separated crate name (`spectra-pyo3`), it can't derive the correct Python module name

**Fix:**

1. Update `crates/spectra-pyo3/Cargo.toml`:
   - Add `name = "spectra_pyo3"` under `[lib]` section (alongside existing `crate-type = ["cdylib"]`)
   - This ensures the compiled output is named `spectra_pyo3.dll`/`.so`/`.dylib`
   - Note: The Python module name is determined by the `#[pymodule]` function name (`_native`), NOT the lib name. But maturin needs the lib name to find and compile the crate.

2. Update `packages/spectra-py/pyproject.toml`:
   - Add `manifest-path = "../../crates/spectra-pyo3/Cargo.toml"` under `[tool.maturin]`
   - This tells maturin where to find the Cargo.toml
   - Keep existing `features = ["pyo3/extension-module"]` and `module-name = "spectra._native"`
   - Update version to `0.2.0` to match the Rust crate version

3. Delete stale `packages/spectra-py/spectra/_native.pyd` — it's from an older build and may not match current exports

<task type="auto">
  <name>Task 2: Fix Python SDK API to match Rust PyO3 exports</name>
  <files>packages/spectra-py/spectra/__init__.py</files>
  <action>
**Problem:** The Python SDK `__init__.py` has API mismatches with the Rust PyO3 module:

1. **Import mismatch:** `from ._native import Agent as _NativeAgent, get_version` — but the Rust module exports `PySpectraAgent` as a class named `PySpectraAgent` (not `Agent`). The `#[pyclass]` struct is `PySpectraAgent`, so in Python it's `_native.PySpectraAgent`, not `_native.Agent`.

2. **Agent.prompt() is async but _NativeAgent.run() is synchronous:** The Python SDK defines `async def prompt()` that calls `self._agent.run(user_input)` — but `PySpectraAgent.run()` is a synchronous blocking function. The async wrapper doesn't actually provide async behavior; it just wraps a blocking call. This blocks the event loop.

3. **Event format mismatch:** `_NativeAgent.run()` returns a JSON string of `[{type: "event", data: {...}}, ...]` (wrapped format), but `Agent.prompt()` does `json.loads(result)` and iterates `events` directly. The wrapped format means each event has `{"type": "event", "data": "..."}` where `data` is a JSON string. The Python code should unwrap this.

**Fix `packages/spectra-py/spectra/__init__.py`:**

1. Fix imports to match actual Rust exports:
   ```python
   from ._native import PySpectraAgent as _NativeAgent, get_version
   ```
   Note: The Rust `#[pymodule]` also exports `create_agent`, `run_agent`, `get_agents`, `delete_agent` as module-level functions. The Python SDK can use either the class-based API or the function-based API. The current code uses the class-based API (`_NativeAgent(json.dumps(config))`), so just fix the class name.

2. Fix `Agent.prompt()` to handle the event format correctly:
   - `PySpectraAgent.run()` returns a JSON string like `'[{"type":"event","data":"{\\"type\\":\\"agent_start\\"}"},...]'`
   - Need to parse outer array, then for each item with `type: "event"`, parse the `data` field as JSON
   - For items with `type: "error"`, use the `message` field directly

3. Consider using `run_streaming()` instead of `run()` for better event handling:
   - `run_streaming()` returns `Vec<String>` directly (no outer wrapper), each element is a serialized `StreamEvent`
   - This is simpler to parse and doesn't have the double-encoding issue
   - But it's also synchronous — for now, accept this limitation and document it

4. Keep the `async def prompt()` signature but make it work correctly with the synchronous underlying call:
   - For now, yield events from the synchronous result
   - In the future, proper async streaming would require PyO3 async support
  </action>
